<html><body><h2>RACER Prot&eacute;g&eacute; Plugin</h2><p>The RACER Prot&eacute;g&eacute; Adapter Plugin enables you to efficently reason aboutontologies within Prot&eacute;g&eacute; 4 using RACER reasoning system provided by<a href="http://www.racer-systems.com/de/index.phtml?lang">Racer Systems</a> <p> Note that the plugin is <b>not</b> shipped with the RACER reasoning system.In order to use it please make sure that you have downloaded RacerPro 2(currently a preview version is freely available at <a href="http://www.racer-systems.com/owlapi">here</a>. <h3>Installation instruction</h3><ol><li>Download the RACER Prot&eacute;g&eacute; adapter plugin.<li>Copy the downloaded file into the plugin folder (typically called <i>plugins</i>) of your local Prot&eacute;g&eacute; directory.<li>Download the RacerPro 2 (resp. the mentioned preview)<li><i>(Optional)</i>Create a directory named <i>racer</i> in the Prot&eacute;g&eacute; directory and copy all Racer binaries there. </ol><h3>Usage</h3>Please make sure that you have an installed RacerPro 2 (resp. RacerPro 2 preview) system on your computer. There are the following possibilities:<ol><li>Make sure that you had followed the installation instructions (1) - (4). RacerPro will then be automatically started and stopped from Prot&eacute;g&eacute; when selecting the RacerPro reasoner. (recommanded)<li>Tell Prot&eacute;g&eacute; where to find RacerPro to start and stop it automatically (also recommanded).<li>Start RacerPro manually. Make sure that you telling Prot&eacute;g&eacute; the port number.</ol>  Take a look at the preference dialog of RacerPro in Prot&eacute;g&eacute;. <h3>Version info</h3><h4>v0.9.1 (23th June 2009)</h4><ul><li>bug for property ranges fixed</li><li>Documentation added</li></ul><h4>v0.9.0 (2nd June 2009)</h4><ul><li>first stable release</li></ul></body></hmtl>
